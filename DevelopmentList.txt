== 모듈 개발 방향 ==

* 꼭 필요할때만 클래스로 구현을 하고, 나머지는 전역변수와 일반함수 작성을 기반으로 자판기 관리 프로그램의 모듈을 구성한다.

1. 알고리즘 개발 : 각각의 탬플릿 클래스로 구현.
2. GUI : 각 윈도우의 UI를 함수 형식으로 개발.
3. 유틸리티 : 함수 형식으로 개발.
4. 관리 프로그램 : 전역 변수를 가지며, 이를 제어할 수 있는 함수를 작성. GUI 이벤트 연결에 용이해야함.

* 개발하는 모듈의 예외처리는 try-except 문을 이용하지 말고, 함수의 리턴문을 이용하여 에러 처리를 할 것.
    만약 함수가 반환을 해야하는 것이 있다면 참조변수를 통하여 그 변수 값을 직접 수정하는 방법으로 리턴문제를 해결할것.
* 소켓 프로그래밍은 추후 강의를 듣고 구현하도록
